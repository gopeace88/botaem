> 수정일: 2025-01-30

## 1. 개요

### 1.1 제품 비전

일반 사용자(비프로그래머)가 보탬e 시스템을 AI의 도움을 받아 쉽게 사용할 수 있는 데스크톱 어시스턴트.

```
"옆에 전문가가 앉아서 화면을 보면서 가이드해주는 경험"
```

### 1.2 문제 정의

| 문제 | 현황 | 영향 |
|------|------|------|
| 시스템 복잡성 | 보탬e 2024.01 개통, 기능 다양 | 일반 회계직원 접근 어려움 |
| 인력 부족 | 비영리단체 인력 한정 | 전문 담당자 배치 어려움 |
| 교육 비용 | 현장 교육 필요 | 시간/비용 부담 |
| 매뉴얼 한계 | 문서 기반 가이드 | 실제 화면과 괴리 |

### 1.3 솔루션

- 실시간 화면 가이드 (여기를 클릭하세요)
- AI 기반 Q&A (이게 뭔가요?)
- 업무 흐름 안내 (다음에 뭘 해야 하나요?)

## 2. 타겟 사용자

### 2.1 Primary Persona: 비영리단체 회계담당자

```
이름: 김회계
나이: 40대
조직: 사회복지법인
역할: 회계/경리 담당 (1인)
IT 수준: 기본 (엑셀, 이메일 정도)

Pain Points:
- 보탬e 시스템이 복잡해서 매번 매뉴얼 찾아봄
- 실수하면 되돌리기 어려움
- 물어볼 사람이 없음 (1인 담당)
- 연말/분기말 정산 시즌에 야근

Goals:
- 실수 없이 업무 처리
- 빠른 업무 완료
- 스트레스 감소
```

### 2.2 Secondary Persona: 신규 담당자

```
이름: 이신입
나이: 30대
조직: 복지관
역할: 신규 회계담당자 (인수인계 받는 중)
IT 수준: 중급

Pain Points:
- 전임자가 제대로 인수인계 안 해줌
- 매뉴얼만으로는 실제 업무 파악 어려움
- 어디서부터 시작해야 할지 모름

Goals:
- 빠른 업무 습득
- 실수 방지
- 자신감 획득
```

## 3. 사용자 스토리

### 3.1 핵심 스토리 (MVP)

#### US-001: 업무 가이드 받기
```
As a 회계담당자
I want to AI에게 "예산 등록하는 법" 물어보면 화면에서 직접 가이드받기
So that 매뉴얼 찾아보지 않고 바로 업무 처리 가능

Acceptance Criteria:
- 자연어로 질문 가능
- 실제 보탬e 화면에서 해당 위치 하이라이트
- 단계별로 안내 (1단계, 2단계...)
- 사용자가 직접 클릭/입력
```

#### US-002: Q&A 질문하기
```
As a 회계담당자
I want to "가수금이 뭔가요?" 같은 용어/개념 질문하기
So that 모르는 것을 바로 해결

Acceptance Criteria:
- 자연어로 질문 가능
- 보탬e 매뉴얼/규정 기반 답변
- 관련 화면/기능 안내 포함
```

#### US-003: 다음 단계 안내
```
As a 회계담당자
I want to "예산 등록했는데 다음에 뭘 해야 해?" 질문하기
So that 업무 흐름을 이해하고 누락 없이 처리

Acceptance Criteria:
- 현재 작업 맥락 이해
- 다음 단계 안내
- 전체 업무 흐름 설명 가능
```

### 3.2 확장 스토리 (Phase 2+)

#### US-004: 자동 실행 모드
```
As a 숙련된 사용자
I want to "예산 등록해줘"라고 하면 AI가 자동으로 클릭/입력
So that 반복 작업 시간 단축

Acceptance Criteria:
- 자동 실행 전 확인 요청
- 각 단계 실행 상황 표시
- 언제든 중단 가능
- 실수 시 되돌리기 안내
```

#### US-005: 오류 해결 도움
```
As a 회계담당자
I want to 화면에 오류 메시지가 뜨면 AI가 해결책 안내
So that 막힘 없이 업무 진행

Acceptance Criteria:
- 오류 화면 감지
- 원인 설명
- 해결 방법 단계별 안내
```

#### US-014: Interactive Watch & Guide (실시간 검증 가이드)
```
As a 회계담당자
I want to 내가 작업을 완료하면 AI가 화면을 확인해서 잘못했으면 다시 알려주기
So that 실수를 바로 잡고 올바르게 업무 진행

Acceptance Criteria:
- 각 단계마다 "완료" 버튼으로 AI에게 검증 요청
- AI가 DOM/스크린샷으로 작업 결과 확인
- 성공 시 다음 단계로 진행
- 실패 시 구체적인 재시도 안내 ("위쪽 버튼을 클릭하세요")
- 연속 실패 시 수동 진행 옵션 제공
- 비용 최적화: DOM 검증 우선, Vision API는 폴백
```

#### US-006: 문서 분석
```
As a 회계담당자
I want to 예산서/결산서 파일 올리면 AI가 분석해서 입력 가이드
So that 수기 입력 실수 방지

Acceptance Criteria:
- 파일 업로드 (엑셀, PDF)
- 내용 분석
- 어디에 뭘 입력할지 가이드
- 분석 완료 후 파일 삭제 (보안)
```

### 3.3 설정 및 관리 스토리

#### US-007: 내 정보 관리
```
As a 사용자
I want to 내 프로필과 조직 정보를 관리
So that 맞춤형 서비스를 받을 수 있음

Acceptance Criteria:
- 프로필 조회/수정 (이름, 연락처)
- 소속 조직 정보 관리
- 비밀번호 변경
- 알림 수신 설정
```

#### US-008: 앱 설정 관리
```
As a 사용자
I want to 앱의 동작 방식을 내 취향에 맞게 설정
So that 편하게 사용 가능

Acceptance Criteria:
- 앱 시작 시 동작 설정
- 테마 설정 (라이트/다크)
- 가이드 속도 조절
- 알림 설정
- 브라우저 설정
```

#### US-009: 멤버십 관리
```
As a 사용자
I want to 내 구독 플랜을 확인하고 업그레이드/다운그레이드
So that 필요한 기능을 사용 가능

Acceptance Criteria:
- 현재 플랜 확인
- 플랜별 기능 비교
- 업그레이드/다운그레이드
- 결제 수단 관리
- 청구 내역 확인
```

#### US-010: 사용량 확인
```
As a 사용자
I want to 내 사용량과 남은 한도를 확인
So that 효율적으로 서비스 사용 가능

Acceptance Criteria:
- 이번 달 사용량 표시
- 플랜별 한도 표시
- 한도 초과 시 알림
- 사용 내역 조회
```

### 3.4 관리자 스토리

#### US-011: 플레이북 관리 (관리자)
```
As a 관리자
I want to 플레이북을 생성/수정/배포
So that 사용자에게 최신 가이드 제공

Acceptance Criteria:
- 플레이북 CRUD
- 버전 관리
- 검토/승인 워크플로우
- 배포 관리
```

#### US-012: Q&A 관리 (관리자)
```
As a 관리자
I want to AI가 생성한 Q&A 답변을 검토/수정
So that 품질 높은 답변 제공

Acceptance Criteria:
- 대기 중인 Q&A 목록
- 답변 수정/승인/반려
- 자주 묻는 질문 관리
- 답변 품질 통계
```

#### US-013: 사용자/조직 관리 (관리자)
```
As a 관리자
I want to 사용자와 조직을 관리
So that 서비스 운영 가능

Acceptance Criteria:
- 사용자 목록/검색
- 사용자 상태 관리
- 조직 관리
- 권한 관리
```

## 5. 멤버십 플랜

### 5.1 플랜 구조

| 구분 | Free | Basic | Pro |
|------|------|-------|-----|
| **월 가격** | 무료 | $30/월 | $65/월 |
| **플레이북** | 기본 10개 | 전체 | 전체 + 커스텀 |
| **Q&A** | 월 50회 | 월 500회 | 무제한 |
| **가이드 모드** | O | O | O |
| **반자동 모드** | X | O | O |
| **자동 모드** | X | X | O |
| **우선 지원** | X | X | O |
| **API 사용량** | 제한 | 표준 | 높음 |

### 5.2 기능 접근 제어

```
Free 플랜:
├─ 기본 플레이북 (회원관리, 기초 업무)
├─ Q&A 월 50회 제한
├─ 가이드 모드만 가능
└─ 광고 표시 (선택적)

Basic 플랜:
├─ 모든 플레이북 접근
├─ Q&A 월 500회
├─ 가이드 + 반자동 모드
├─ 이메일 지원
└─ 광고 없음

Pro 플랜:
├─ 모든 기능 + 커스텀 플레이북 요청
├─ Q&A 무제한
├─ 모든 모드 (자동 포함)
├─ 우선 지원 (채팅)
├─ 사용량 분석 리포트
└─ 팀 기능 (Phase 3)
```

### 5.3 결제 방식

| 방식 | 설명 | 수수료 |
|------|------|--------|
| 신용/체크카드 | Stripe 연동 | ~3% |
| 계좌이체 | 한국 은행 연동 | ~1% |
| 세금계산서 | 법인/단체용 | 별도 |

### 5.4 무료 체험

```
Pro 플랜 14일 무료 체험
├─ 신규 가입 시 자동 적용
├─ 결제 정보 등록 불필요
├─ 체험 종료 후 Free로 자동 전환
└─ 체험 중 유료 전환 시 할인 (20%)
```

## 7. 제약사항

### 7.1 기술적 제약

- 보탬e 시스템 화면 구조 변경 시 플레이북 수정 필요
- 브라우저 자동화는 사용자 PC에서만 실행 (클라우드 불가)
- AI 응답 품질은 Claude API 성능에 의존

### 7.2 비즈니스 제약

- 월 $30-65 구독료 내에서 API 비용 관리 필요
- 현장 A/S 없이 원격 운영 가능해야 함
- 보탬e 시스템 변경에 빠르게 대응해야 함

### 7.3 법적 제약

- 사용자가 직접 로그인/조작 (대행 불가)
- 업무 데이터 수집/저장 금지
- AI 오류 면책 조항 필요

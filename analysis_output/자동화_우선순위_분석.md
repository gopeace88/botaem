## 자동화 후보 상세 분석

### 🥇 1순위: 전자세금계산서 집행등록 자동화

**화면**: 집행등록 (전자세금계산서)

#### 현재 프로세스
```
1. 홈택스에서 전자세금계산서 발행
2. 보탬e 집행등록 화면 접속
3. 전자세금계산서 조회 (홈택스 연동)
4. 거래처 정보 확인/입력
5. 비목/세목 선택
6. 집행금액 입력
7. 집행요청
```

#### 자동화 방안
```
[홈택스 API] ──자동조회──▶ [보탬e 집행등록]
      │                         │
      ├─ 세금계산서 목록        ├─ 거래처 자동매칭
      └─ 거래내역 정보          └─ 비목/세목 규칙 기반 자동선택
                                 └─ 일괄 집행요청
```

#### 평가 점수
| 기준 | 점수 (10점 만점) | 사유 |
|------|-----------------|------|
| 반복성 | 10 | 매 거래마다 동일 작업 반복 |
| 대량처리 | 9 | 월 수십~수백 건 처리 |
| 오류감소 | 9 | 금액, 거래처 입력 오류 방지 |
| 기술 실현성 | 8 | 홈택스 연동 이미 구현됨 |
| 시간절감 | 9 | 건당 5분 → 1분 이하 |
| **종합** | **9.1** | **최우선 자동화 대상** |

#### 예상 효과
- 수작업 90% 감소
- 입력 오류 0%
- 월 처리시간 80% 단축

### 🥈 2순위: 카드사용내역 집행등록 자동화

**화면**: 보조금전용카드 사용내역관리 (97003), 집행등록 (신용카드)

#### 현재 프로세스
```
1. 전용카드 사용 (거래 발생)
2. D+1일 카드사 데이터 연동 (야간 배치)
3. 보탬e에서 카드사용내역 조회
4. 각 내역별 집행등록 화면 이동
5. 비목/세목 선택
6. 집행요청
```

#### 자동화 방안
```
[카드사 배치 데이터] ──야간연동──▶ [보탬e]
                                    │
                                    ├─ 미등록 내역 자동 식별
                                    ├─ 비목/세목 자동 매핑 (규칙 기반)
                                    └─ 일괄 집행등록/요청
```

#### 평가 점수
| 기준 | 점수 (10점 만점) | 사유 |
|------|-----------------|------|
| 반복성 | 10 | 카드 사용 시마다 등록 필요 |
| 대량처리 | 8 | 월 수십 건 |
| 오류감소 | 8 | 비목 선택 오류 방지 |
| 기술 실현성 | 9 | 데이터 이미 연동됨 |
| 시간절감 | 8 | 건당 3분 → 자동 |
| **종합** | **8.6** | **2순위 자동화 대상** |

### 🥉 3순위: 집행이체 일괄처리 자동화

**화면**: 집행이체관리 (93003)

#### 현재 프로세스
```
1. 집행요청 완료 건 확인
2. 집행이체관리 화면 접속
3. 이체대상 선택
4. 이체비밀번호 입력
5. 인증서 인증
6. 이체 실행
```

#### 자동화 방안
```
[집행요청 완료건] ──스케줄링──▶ [일괄이체]
                                │
                                ├─ 특정 시간 자동 실행
                                ├─ 이체비밀번호 자동 입력
                                └─ 인증서 인증 (반자동)
```

#### 평가 점수
| 기준 | 점수 (10점 만점) | 사유 |
|------|-----------------|------|
| 반복성 | 9 | 일/주 단위 반복 |
| 대량처리 | 9 | 다건 일괄 이체 |
| 오류감소 | 7 | 이체 누락 방지 |
| 기술 실현성 | 5 | 인증서 인증 필요 (반자동) |
| 시간절감 | 8 | 건당 2분 → 일괄 처리 |
| **종합** | **7.7** | **인증서 이슈로 반자동화** |

#### 제약사항
- ⚠️ 공동인증서 비밀번호 입력 필요
- ⚠️ 완전 자동화 어려움 (보안 이슈)
- 💡 특정 시간에 일괄 실행 후 인증만 수동

### 4순위: 이자/수익금 자동등록

**화면**: 집행마감 (94001) - 이자등록, 수익금등록

#### 현재 프로세스
```
1. 집행마감 화면 접속
2. 예치계좌 이자 확인 (은행 조회)
3. 이자등록 버튼 클릭
4. 이자금액 입력
5. 저장
```

#### 자동화 방안
```
[예치계좌] ──잔액/이자조회──▶ [보탬e]
                               │
                               └─ 이자 자동계산 및 등록
```

#### 평가 점수
| 기준 | 점수 (10점 만점) | 사유 |
|------|-----------------|------|
| 반복성 | 6 | 정산 시점에만 발생 |
| 대량처리 | 4 | 사업당 1건 |
| 오류감소 | 9 | 이자 계산 오류 방지 |
| 기술 실현성 | 7 | 계좌 조회 API 필요 |
| 시간절감 | 6 | 단순 작업이나 정확성 중요 |
| **종합** | **6.4** | **오류 방지 효과 높음** |

### 5순위: 정산검토 자동 분류

**화면**: 검토대상목록조회 (지자체), 정산검토 의견등록

#### 현재 프로세스
```
1. 지자체 담당자가 집행건 목록 조회

```

#### 자동화 방안
```
[집행건 목록] ──규칙검증──▶ [자동분류]
                            │
                            ├─ 정상건: 자동 "검토완료"
                            ├─ 이상건: "확인필요" 플래그
                            └─ 위반건: "보완요청" 자동
```

| 대량처리 | 7 | 사업당 수십 건 |
| 오류감소 | 5 | 판단 영역 있음 |
| 기술 실현성 | 4 | 규칙 정의 복잡 |
| 시간절감 | 7 | 단순건 자동 처리 |

### 6순위: 실적보고서 자동생성

**화면**: 실적보고서 등록 (94008)

#### 현재 프로세스
```
1. 정산마감 완료 후
2. 실적보고서 화면 접속
3. 기본현황, 계획대비실적, 사업성과 등 입력
4. 정산보고서 탭 (자동생성됨)
5. 보고서 제출
```

#### 자동화 방안
- 정산총괄표, 집행내역은 이미 자동생성
- 기본현황, 사업성과는 수동 입력 필요 (판단 영역)
- 💡 템플릿 기반 초안 자동생성만 가능

#### 평가 점수
| 기준 | 점수 (10점 만점) | 사유 |
|------|-----------------|------|
| 반복성 | 5 | 사업 종료 시 1회 |
| 대량처리 | 3 | 사업당 1건 |
| 오류감소 | 4 | 내용 작성은 수동 |
| 기술 실현성 | 3 | 성과 평가는 사람 판단 |
| 시간절감 | 4 | 일부만 자동화 가능 |

## 자동화 구현 로드맵

### Phase 1: Quick Win (1-2주)
- ✅ 카드사용내역 집행등록 자동화
  - 이미 연동된 데이터 활용
  - 비목/세목 매핑 규칙 정의
  - 일괄 집행등록 스크립트

### Phase 2: Core Automation (3-4주)
- ✅ 전자세금계산서 집행등록 자동화
  - 홈택스 연동 활용
  - 거래처 자동매칭 로직
  - 집행요청 배치 처리

### Phase 3: Enhancement (5-6주)
- ⚠️ 집행이체 일괄처리 (반자동)
  - 이체대상 자동 선택
  - 인증서 인증만 수동

### Phase 4: Advanced (7-8주)
- 💡 이자/수익금 자동등록
- 💡 정산검토 규칙 기반 사전분류

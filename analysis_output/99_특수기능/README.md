## [보탬e]가수금 처리방법 안내_매뉴얼_v1.0.docx

**전체 길이**: 1,384자

### 주요 키워드
- 이체: 6회
- 조회: 5회
- 보조금: 5회
- 버튼: 5회
- 집행: 4회
- 확인: 4회
- 교부: 3회
- 선택: 3회
- 입력: 3회
- 클릭: 3회

### 입력 필드
내역을, 으로, 입금계좌번호, 입금정보를

### 내용 요약
```
보조금 전용계좌로 입금되는 정상적인 방식
-. 보조금 교부 : 지자체 담당자가 e호조+ 를 통해 연계지출로 보탬e로 보조금 교부
(내부적으로 어느 보조사업자에게 입금되는지 연결정보가 있음)
-. 보조금 집행취소 : 예치형 사업 보조사업자가 집행처리 했던 금액을 취소하기 위해 다시 입금하는 경우 보탬e 시스템을 통해 집행취소 하면 보조사업자의 자부담 계좌에서 보탬e 계좌로 입금됨 (보탬e 를 통해 입금하기 때문에 어느 보조사업자가 입금하는 것인지 연계정보가 있음)
-. 정산반환 : 보조사업자가 사업수행 종료 후 잔액 및 이자금액을 보탬e 로 반환하는 경우 보탬e 시스템을 통해 반환이체 하면 자부담 계좌에서 보탬e 계좌로 입금됨
(보탬e 를 통해 입금하기 때문에 어느 보조사업자가 입금하는 것인지 연계정보가 있음)
가수금 이란?
-. 보탬e 계좌로 입금된 금액은 있는데 누가 무슨 용도로 입금하였는지 알 수 없는 금액
-. 보조사업자가 집행취소 또는 정산반환을 할때 보탬e 를 통하지 않고 입금하면 누가 무슨 용도로 입금하였는지 연결 정보가 없기 때문에 가수금으로 처리됨.
-. 보조금 교부시 e호조에서 연계지출로 처리 하지 않는등 정상적인 방식이 아닌 방법으로
보탬e 전용계좌(예치계좌) 로 입금하는 경우 누가 무슨 용도로 입금하였는지 연결 정보가 없기 때문에 가수금으로 처리됨.
가수금 처리방법
-. 우선 누가 입금하였는지를 알아야 함.
누가 입금하였는지 알 수가 없는 경우 되돌려 줘야 하는데 누구에게 돌려줘야 하는지 알지 못하면 처리할 수 없음.
-. 누가 무슨 용도로 잘못 입금한 내용을 알게되면
관리자 출금 기능을 ...
```
